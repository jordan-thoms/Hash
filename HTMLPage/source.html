MD5 Hash Cracking using MapReduce

MD5 is a one-way hash algorithm that underpins the security of many website's authentication (e.g. think login/logout). Websites MD5 hash passwords and store it in their databases. Cracking the password requires some form of compute intensive brute-force. However, if the task was distributed over a cluster of servers using a MapReduce then the feasibility of a brute-force increases.

MD5 Hash Algorithm in detail

What is MD5?

The basic concept of MD5 hashing is not new, a one-way hashing algorithm takes an input message of arbitrary length and hashes it to output a uniquely identifying 128-bit digest. This means that no other (even slightly altered) input should generate the same hash digest. When there are two or more different inputs that hash to the same digest, they are known as collisions. Though, by pigeonhole principle, two input messages could theoretically hash to one digest, it’s practically infeasible to do so. In which case, a 'good' one-way hashing algorithm should be one that avoids collisions as much as possible. Although MD5 has shown to be exploitable by using collisions by Wang and Yu [1], it’s still widely used.

Where is it used?

The MD5 hash algorithm underpins the security of many websites because of its cryptographic features and performance. It enables websites to leverage it to perform authentication. This is achieved by saving the MD5 hash of the original password in the site’s database. To authenticate, the hash of the user-entered password is computed and compared against the original hash. If the hashes are the same, then the user has entered in the original password and can be authenticated.

Why is it being used?

As websites use MD5 in its databases, hackers tend to want to reverse the hashes in order to gain access to additional privileges. Despite hackers, website developers typically argue that because the computational cost of reversing the password (in polynomial time) is so high, it would be infeasible to ‘crack’ a MD5 hash to obtain the original password. There are up to 57 billion (62^6) combinations for a six-character alphanumeric password, or 3.5 trillion for seven character passwords. Assuming a cracking rate of 1 million hashes per second, for six-characters, it would take up to 16 hours to go through all possible combinations. For eight-characters, it would take 7 years. This is because the problem complexity increases by 62 times, as the password increases by one character.

MapReduce Architecture in a Nutshell

What is a MapReduce

MapReduce is a programming model for processing large data sets, and the name of an implementation of the model by Google. MapReduce is typically used to do distributed computing on clusters of computers. The model is inspired by the map and reduce functions commonly used in functional programming, although their purpose in the MapReduce framework is not the same as their original forms. MapReduce libraries have been written in many programming languages. A popular free implementation is Apache Hadoop.

Instructions to execute it

Servers

You need to set up a cluster of servers. This brief tutorial will not teach you how to set it up, but you should at least have password-less SSH set up on your cluster of servers. It is best to set up Juju.

Running it

To spin up server cluster using Juju

./HadoopHash/create_cluster.sh

Juju will spin up servers with specific AMI on Amazon EC2 Instances.

To deploy the application to the server:

./HadoopHash/deploy_to_server.sh IP_ADDRESS

Replace IP_ADDRESS with an actual IP address or Domain name. E.g. (192.168.1.1)
